# 旧系统数据迁移字段映射模板

本文档用于一期旧系统数据迁移预留能力。当前版本只定义标准 CSV/Excel 模板、字段映射和校验规则，不连接真实旧系统，不执行正式入库导入。

## 使用约定

- 文件格式预留支持：`.csv`、`.xlsx`。
- Excel 工作表建议命名：`users`、`referrals`、`levels`、`assets`、`orders`、`products`。
- 所有金额、资产数量使用十进制文本，后端按 `BigDecimal` 处理。
- 旧系统主键必须保留在 `old_*_id` 字段中，正式导入时用于幂等、追溯和异常报告定位。
- PRD 标记待确认的业务规则不得在迁移中写死，只能落到配置说明或待确认备注。

## 一期校验项

- 普通用户无上级：非种子账号必须存在 `inviter_old_user_id`。
- 重复手机号：`phone` 非空时批次内不可重复。
- 重复身份证：`id_card_no` 非空时批次内不可重复。
- 异常等级：`level_code` 必须匹配目标系统 `UserLevel`。
- 负资产：`available_amount`、`frozen_amount` 不允许小于 0。
- 推荐链断裂：`inviter_old_user_id` 必须指向本批次用户。
- 推荐链循环：任意用户沿上级链路不可回到自身或已访问节点。

## 后台入口预留

- 校验接口：`POST /api/v1/admin/migration/import/validate`
- 异常报告导出预留：`GET /api/v1/admin/migration/import/reports/{reportId}/export`
- 后台页面一期可仅展示入口说明、模板下载说明和“上传后先校验，不直接入库”的安全提示。
