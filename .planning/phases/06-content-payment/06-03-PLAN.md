  ```java
  @GetMapping
  public Result<IPage<ContentResponse>> getContent(
          @RequestParam(defaultValue = TYPE_VIDEO) String type,
          @RequestParam(required = false) String keyword,  // NEW
          @RequestParam(defaultValue = "1") Integer page,
          @RequestParam(defaultValue = "10") Integer size) {
      Long userId = currentUserId();
      String likePattern = keyword != null && !keyword.isBlank() ? "%" + keyword + "%" : null;

      if (TYPE_ARTICLE.equals(type)) {
          LambdaQueryWrapper<HelpArticle> wrapper = new LambdaQueryWrapper<HelpArticle>()
              .eq(HelpArticle::getStatus, 1);
          if (likePattern != null) {
              wrapper.like(HelpArticle::getTitle, likePattern);  // D-SEARCH-02: title-only
          }
          wrapper.orderByDesc(HelpArticle::getCreatedAt);  // D-SEARCH-03: recency sort
          IPage<HelpArticle> articles = helpArticleMapper.selectPage(new Page<>(page, size), wrapper);
          return Result.success(articles.convert(a -> articleResponse(a, userId)));
      }

      LambdaQueryWrapper<Video> wrapper = new LambdaQueryWrapper<Video>()
          .eq(Video::getStatus, 1);
      if (likePattern != null) {
          wrapper.like(Video::getTitle, likePattern);  // D-SEARCH-02: title-only
      }
      wrapper.orderByDesc(Video::getCreatedAt);  // D-SEARCH-03: recency sort
      IPage<Video> videos = videoMapper.selectPage(new Page<>(page, size), wrapper);
      return Result.success(videos.convert(v -> videoResponse(v, userId)));
  }
  ```

  Key implementation notes:
  - D-SEARCH-01: Search covers videos and help articles only (endpoint already filters by type)
  - D-SEARCH-02: Search matches title field only using SQL LIKE
  - D-SEARCH-03: Results sorted by created_at desc (recency)
  - D-SEARCH-05: No minimum character check - empty keyword returns all content
  - D-SEARCH-06: Empty results handled by existing UI (shows "暂无内容")
  - T-6-03: MyBatis-Plus parameterized queries prevent SQL injection
